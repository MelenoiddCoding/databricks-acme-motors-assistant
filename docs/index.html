Databricks · Foundation Model · Function Calling · Delta Tables

Acme Motors Dealership Assistant

A Databricks learning portfolio that uses a foundation model as a function router. The LLM selects safe business functions, Python executes controlled queries, and Delta Tables provide the source of truth.

View End-to-End Notebook View GitHub Repo

Business Problem

A dealership manager wants to ask sales questions in natural language, such as: “How many F150 trucks did I sell this month?” The assistant must not hallucinate numbers or generate arbitrary SQL.

Architecture

User question
↓
Databricks Foundation Model via ai_query
↓
Structured JSON function call
↓
Python FunctionRegistry
↓
Safe Spark SQL query
↓
auto_sales_transactions Delta Table
↓
Final answer + audit log

Key Design Decision

The LLM routes. Python executes. Delta is the source of truth.

The model does not calculate sales numbers, does not generate SQL, and does not access arbitrary tables. It only selects from approved functions.

Notebook Walkthrough

01 · Auto Sales Delta Table Create mock sales data, schema, quality checks, and Delta table. 02 · Manual Business Queries Validate SQL queries before converting them into safe functions. 03 · Function Catalog Define allowed functions, arguments, examples, and JSON outputs. 04 · Function Registry Build the Python object that validates and executes functions. 05 · Foundation Model Router Use Databricks ai_query to convert questions into function calls. 06 · End-to-End Assistant Connect router, registry, Delta query, answer, and audit log.

Example Interaction

User question:

How many F-150s have I sold this month?

Foundation model router output:

{
  "function_name": "get_product_sales_summary",
  "arguments": {
    "product_model": "F150",
    "date_range": "this_month"
  }
}

Final answer from Delta Table:

For this_month, F150 sold 5 units with $266,500.00 total revenue and an average sale price of $53,300.00.
